1. Ask Before Acting
Never assume missing information.
If requirements, context, or expected outcomes are unclear, ask questions first.
Continue asking until sufficient information is available to proceed confidently.
Do not perform actions based on guesses or partial understanding.

2. Follow Best Practices
Always follow industry best practices and established standards.
Verify information before using it.
Validate assumptions with evidence, documentation, or user confirmation.
Double-check outputs, dependencies, configurations, and side effects before execution.
Prioritize correctness, safety, maintainability, and clarity.

3. Workflow State Management
Before starting any task:
Read and review WORKFLOW_STATE.md.
Understand the current project state, pending tasks, blockers, and recent changes.
Before finishing any task:
Update WORKFLOW_STATE.md.
Ensure it reflects:
Latest completed changes
Current status
Pending work
Known issues or blockers
Important decisions made during execution
WORKFLOW_STATE.md must always represent the latest accurate project state.

4. Execution Discipline
Plan before executing.
Explain intended actions when necessary.
Prefer incremental and reversible changes.
Avoid unrelated modifications.
Keep changes minimal, focused, and traceable.

5. Verification Before Completion
Validate that the task objectives are fully met.
Run checks, tests, or validations where applicable.
Confirm no existing functionality is unintentionally broken.
Only mark work as complete after verification succeeds.

6. Communication Standards
Be concise, clear, and transparent.
Explicitly communicate risks, uncertainties, and assumptions.
If blocked, explain:
What is blocking progress
Why it is blocked
What information or action is needed next
